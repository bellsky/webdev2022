당연히 대면 강의.
정보보안, 컴퓨터일반, 프로그래밍언어.

-정보보안
    : 대칭키 암호 알고리즘에 대해 복습 및 추가 학습하였다.

    -전수공격
        :모든 키 대입
        확률적으로 절반쯤 대입했을 때 정답을 맞춘다.
        56비트(7글자)까지 요즘 기술로 금새 뚫린다.


    -대칭블록암호 설계요소
        -블록길이 : 주로 64비트. 요즘은 더 높음.
        -키길이 : 보편적 128비트. 길이가 길수록 보안업, 성능다운.
        -라운드수 : 전형적 16라운드
        -서브키 생성 알고리즘 : 복잡할수록 암호해독이 어려워진다.
        -라운드 함수 : 복잡할수록 암호해독이 어려워진다.

    -대칭블록암호 설계시 고려 사항
        -빠른 소프트웨어 암호화 복호화
        : 암호화에 오래걸리면 사용성 저하
        -용이한 해독
        : 알고리즘을 간결하고 명확하게 설명할 수 있어야 한다. 
        취약성 분석이 쉬워지므로 개선 또한 쉬워진다.

    DES=56비트
    AES= 128비트, 256비트, 192비트, 256비트
    3중DES=168비트 (56*3)

    -DES
        -알고리즘은 DEA라고 부른다. 암호화 방식(포준)이름이 DES.

        평문 64비트
        키 56 비트
        구조 FEISTEL 네트워크 변형
        라운드 16
        서브키 원래키로부터 16개 48비트 서브키 생성하여 각 라운드에서 사용


    -DES의 강도
        -키길이의 취약성. 
        56비트 키이므로, 마이크로 초 당 천의 암호화 수행 시 
        매우 오랜 시간이 걸렸었지만 요즘은 컴퓨터가 더 빨라져서 약해짐.

    -DES 크래커 머신 1938년 by EEF
        -25만불 이하로 만들어진 특별기계.
        -공격이 3일 이내로 성공함.
        -그래서 더 이상 쓰지 않음.
        -최근 멀티코어 컴퓨터는 초당 10^9의 키조합 가능.
        슈퍼컴 기준 10^13번 가능. 2012년 자료로.

    -AES 암호화의 경우 가장 낮은 128비트마저 양자컴퓨터로도 해독에 매우 오랜 시간이 걸린다.


    -3중 DES
        :3개의 키를 사용해 dES알고리즘 3번 수행.

        암호-복호-암호 순서로 수행.

        암호화 
        C= E(K3D(K2E(K1P)))

        C=암호문
        P=평문
        E[K,X] 키 K를 이용해 X를 암호화
        D[K,Y] 키 K를 이용해 Y를 복호화

        사용한 이유 -> 기존 DES와의 호환성 때문에.

        3개의 다른 키를 사용하여 168비트의 유효성 길이.
        1개의 같은 키를 이용할 경우 DES와 동일 알고리즘이 된다. DES암호화복호화 = 1 이므로.

    -3DES이 주요 약점
	    -3DES는 느리다.
	    -64비트 블록을 사용하기에 블록 사이즈가 커져야한다.
        -따라서 장기적 표준으로 DES는 부적합하다.
        -NIST는 1997년에 3DES와 같거나 더 나은 강력한 보안성을 갖으면서 
        월등히 개선된 효율적인 새로운 AES (Advanced Encryption Standrd)를 공모함.

        최종선정 = 2001년 11월 최종 표준.
        레인달이 최종 표준으로 선정.
        개발자 : 벨기에 암호학자 요안 대믄, 빈슨트 레이믄.

    이외에는 실습은 별로 없을 것. 인공지능을 위한 기초 지식이지만, 
    인공지능 부분을 세부적으로 들어가진 않을 예정.

    -컴퓨터의 역사
    -컴퓨터 하드웨어의 발전 역사
    -컴퓨터의 종류

    -컴퓨터 기술의 부정적인 요소

-프로그래밍 언어론

    : 지난 시간의 연장.
    프로그래밍 언어의 역사에 대해 마저 배움. 

    -1950년대 중반
     포트란 = 포뮬러 트랜슬레이션 약어.
        최초의 고급 언어. 효율적인 기계어 코드로 생명이 길었다.
        배열, 반복구조, 분깆문, 부트프로그램 등의 특징이 이후 언어에 반영됨.

    -1950년대 후반
        코볼 = 상업자 자리 처료 용. 이것도 생명이 길었다.
        알골로 60 = 숫자에 따라 버전업됨.
        리스프 = 리스트 프로세서의 약어. 기호연산, 리스트 처리용. 함수처리에 적합. 로봇에 쓰였다. 인공지능 초기 언어라고 부르기도. ‘폰노이만 아키텍쳐 탈피’를 위해 고안됨.
        APL = 배열처리용.


    -1960년대
        PL/1 = IBM 프로젝트. 360 시리즈에 사용할 계획이었으나 문제점이 많았다. 새로운 개념인 병행성, 기억 장소 할당, 예외처리가 도입됨.

        Algol68 = 60 기반 여러 기능 추가. 일관된 구조로. 직교성을 강조함. 대신 새로운 용어를 과다 사용함. 범용 컴퓨터에서 사용이 불가능하다는 문제가 있었음.

        Basic. 시분할용ㅇ의 간결한 언어. 교육용, 사무처리용, 가정용으로 쓰였다. ANSI 표준. 이후의 베이직에서 제어구조와 변수 선언, 프로시저가 추가되었다.

        Snobol : 최초의 문자열 처리 언어. 패턴 매칭 기능이 다양함.

        Simular67 : Simula1+albol60. 시뮬레이션 목적. 클래스 개념 최초 도입의 객체 지향 언어.


    -1970년대 언어의 특징. 간결성, 추상화, 연구사항.
        파스칼, C, C파스칼.


    -1970년대 중후반.
        CLU
        Euclid
        Mesa


    -1980년대 특징. 개념통합 Ada 등장. 함수형 언어 관심 부여. 논리형 프로그래밍 언어 Prolog 등장. 객체 지향 언어 활발한 연구 및 개발.

        Ada : 미국방성에서 독자적 사용.

        Modula-2 : 소규모. 임베디드 목적.


        Prolog : 논리형 언어. 인공지능 분야에서 많이 사용.

        SETL : 미적분 , 수학의 교육 모델. 구현 결핍 존재.

    -80년대 함수형 언어들 등장.


    -90년대 언어의 특징.
        :제 4세대 언어. 응용문제 빠르게 구현, 
        실사용자가 직ㅈ버 프로그래밍, 데이터베이스를 쉽게 처리. HTML과 java 등장.

        java = C++ 기반. 간결성과 신뢰성 제공 원칙. 자료형과 클래스 제공. 
        기본 자료형은 스칼라형. 배열은 미리 정의된 클래스 객체로 제공된다.(C++와 다른 점)
