    9.1 데이터 통신의 개요

        1. 데이터 톹신의 개념
            : 멀리 떨어져 있느 송신자 컴퓨터와 수신자 컴퓨터 또는
            입출력 장치를 통신 회선으로 연결해 데이터를
            처리하고 전송하는 것.

            -데이터 통신 기술 : 데이터 통신이 원활히 이루어지도록
            데이터를 전송하고 처리하는 기술.

            -데이터통신의 특징
                : 고속화 대용량화 모바일화

        2. 데이터 통신의 발전 과정
            -전신(Telegraph)
            -정보부호화=모스부호기
            -전화기=그레이엄 벨   -> 토마스 에디슨의 탄소 송화기
            -하인리히 헤르츠가 전자파를 발견 
                    -> 20세기 초 전자파를 이용해 모스부호를 무선으로 전송

            -최초의 컴퓨터 네트워크는 미국에서 개발된 반자동 방공망 시스템
            -최초의 패킷 교환망 미국방성 아르파 넷 = WWW의 시초
            -이후로 TCP/IP 프로토콜 기반 인터넷 시대 개시


    9.2 네트워크의 개요

        1. 프로토콜 
            : 컴퓨터 네트워크에서 데이터를 주고받을 때 수행되는 절차.
            -데이터 전송을 위한 두가지 요건 : 하드웨어적 연결 + 프로토콜
            -프로토콜의 종류
                -FTP
                -HTTP
                -TCP/IP



        3. OSI 7계층 모델
            : 국제 표준화 단체인 ISO가 제안.
            네트워크 아키텍쳐 간 호환이 안되는 문제를 해결하기 위해 만들어짐.


            1계층 물리 - 데이터링크 - 네트워크 - 전송 - 세션 - 표현 - 7계층 응용


        4. 데이터 전송의 유형

            단방향
            양방향
            반이중


            회선 접속 방식에 따라
            점대점 다자점


        5. 데이터 전송 방식

            직렬 전송 : 하나의 통신 회선. 동기식/비동기식

            병렬 전송 : 비트 블록을 한 번에 전송. 
                비트블록 = 하나의 문자를 표시하는 데 필요한 비트 수.
                거리가 짧고 고속 전송이 필요한 곳에 쓰인다.


        6. 데이터 전송 장치

            모뎀 : 컴퓨터나 단말기 등의 데이터 통신 기기를 통신 회선에 접속시켜주는 장치.
                컴퓨터의 디지털 신호를 아날로그 신호로 변환해주고,
                    데이터를 수신할 때에는 아날로그 신호를 디지털 신호로 복구함.

    CH.7 자료형

        구조자료형
            : 여러 자료를 묶어 하나의 단위로 처리하는 자료형
            -array = homogeneous data set
            -record = heterogeneous set ->이질성. 다른 자료형들을 모을 수 있는 구조자료형.

            -배열 array
                이름, 차원, 원소형, 첨자 집합의 형과 범위로 구성됨.
                첨자 = 일반적으로 연속적인 정수형 집합.
                (Pascal에서는 스칼라형 자료형. 실수형 불가능.)
                각 차원 : 상한 <= 하한  -> 크기 = 상한-하한 +1

                -배열 첨자의 경계값
                 = 배열에서의 상한/하한값 표현. 
                    우리가 쓰는 언어에서는 0에서 시작하는 게 다수.
                    Fortran은 1부터 시작하는 상수. Pascal에서는 크기가 달라도 다른 자료형.
                    Visual Basic에서 기본 생성 시 크기는 0~100까지 101의 크기

            
            -배열 사용의 예


            -배열 자료형과 고려 사항들
                배열 이름과 배열 원소에 대한 구문
                원소 값에 대해 어떤 자료형이 쓰이는지
                첨자로 어떤 자료형을 쓸 수 있는지
                배열 크기와 바인딩 시간
                배열 이름에 대한 주소 결정이 얼마나 복잡한지
                어던 형태의 슬라이싱을 제공하는지
                배열 초기화 시키려면 어떤 종류의 문장이 허용되는지
                배열에 대한 내정된 연산은 어떤 종류가 허용되는지


            -구조 자료형의 역사
                COBOL에서 시작됨.
                PL/L과 Pascal의 record에서 가변부 추가.
                레코드 내의 필드 참조 방식
            
            -레코드 초기화

        포인터자료형


            포인터 : 객체에 대한 참조
            포인터 변수 : ㄷ객체를 참조하기 위한 주소를 값으로 취하는 식별자.
            필요성 : 실행 시간까지 크기를 알 수 없는 자료(동적할당)
                    , 다중 관계
            
            문제점 
                -다수의 포인터가 동일 객체를 지시할 때 : 다른 이름 사용
                -포인팅 되지 않은 객체가 존재할 때 : 현수 참조 문제
                -포인터 변수 r-value (=객체를 가리킴) / l-value
                            r= real    l = location
                

            C/C++의 포인터
                어셈블리어와 같이 사용 가능
                    -> 유연성 제공, 조심스러운 사용 필요.
                    -> 포인터의 산술 연산 가능.
                    -> 메모리의 모든 기억장소 내의 변수에 대한 모든 위치 지정이 가능
                    
                    ' ο 'dereferencing = 값을 가져오는 것.
                    ' & ' 주소를 지정

            자료형 변환
                묵시적 변환 = 수식 평가나 배정 시 발생.

                명시적 변환 = cast 연산자 사용

                종류
                    -축소변환 -> 절단/반올림 발생

                    -확대변환 -> 데이터 손실 없음.

                스칼라형 : 자료형 반환이 ?역

            
            자료형 동치

                일반적인 규칙
                    이름 동치
                        -> 함께 선언되거나 또는 동일 식별자 이름으로 선언되면 동일 자료형으로 간주.

                    구조 동치 
                        -> 구성 요소가 모든 측면에서 같으면 동일 자료형으로 간주
                    
                    선언동치 
                        -> 이름 동치에서 함께 선언된 것만을 분리하고자 할 때 사용

                    
                    알골68,알골w,파스칼이 자료형 정의에 중요 공헌.
                        대신 동치와 적법성의 문제를 동시에 가짐.

                    자료형의 적법성? 객체 형이 특정 문맥에서 정당한지 결정하는 의미적인 규칙
