-프로그래밍언어론


    ch4 프로그래밍 언어의 구문과 구현 기법

    언어 문법 = 구문론(문법적으로 옳은지) + 의미론(의미가 성립하는지) 
    = (syntax +semantics)  문법은 크게 두가지 카테고리로 나뉜다. 


    -언어 구문 - 프로그래밍 언어의 기본 문자 집합.

        -정합 순서 : 문자 또는 문자열에 대한 일반적인 순서 언어.
        언어 구현시에 결정하며 일반적으로 문자의 vit 조합 표현 순서에 영향을 준다. 프로그래머가 정의할 수 있다.


        -어휘 구조 용어.
        어휘 토큰 = 요소 = 단위

        -식별자, 미리 정의 된 식별자, 예약어.
        장점 : 판독성 증가. 컴파일러가 기호 테이블을 빠른 시간에 탐색 가능.
        단점 : 예약어가 많아질수록 기존 식별자와 중복 우려.

        -구분자, 분리자
            blank, 세미콜론 등. 


    즉 문법이란, 정해진 토큰을 정해진 순서로 배열한 게 syntax, 의미가 부여됐을 때 semantics까지 만족하게 된다.



        문법에는 4가지 종류가 있음. 놈 촘스키 씨가 분류함. -> 5장 내용.

        1레벨 : 무제약적 문법.
        2레벨 : 컨텍스트(문맥) 센서티브(감각적) 문법
        3레벨 : 컨텍스트 프리 문법   -> 기본 : 자료구조에 stack이 반드시 있어야함. 
        4레벨 : 레귤러 (정규) 문법


        Syntax의 3가지 표기법.
            -BNF 표기법.
                : 구문 형식을 정의하는 가장 보편적인 표기법 
                한 언어의 구문에 대한 BNF 정의 = 언어의 문장을 생성하는 생성 규칙 정의.

                생성 규칙 : 생성 규칙의 왼쪽에는 정의 대상, 오른쪽에는 정의 표현. 

            -Extend BNF  EBNF 표기법.
                :

            -Systax Diagram 구문 도표

            -Parse Tree 파스 트리
